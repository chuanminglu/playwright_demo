# 实战演练：LLM 驱动的 Playwright 自动化测试

 **目标** ：掌握如何结合大语言模型（Claude/DeepSeek/ChatGPT）与 Playwright，高效完成基于 [Swag Labs](https://www.saucedemo.com/) 的端到端 (E2E) 测试开发。

 **核心理念** ：从“手写代码”转变为“设计逻辑 + 审查代码”。

---

## 📅 演练流程大纲

### 第一阶段：环境准备 

1. 新建项目文件夹。
2. 使用 `setup.js` 脚本一键生成项目骨架（跳过繁琐配置）。
3. 安装依赖：`npm install` 和 `npx playwright install`。

### 第二阶段：AI 辅助生成 POM

* **任务 1** ：生成 `LoginPage.ts`（基础表单处理）。
* **任务 2** ：生成 `InventoryPage.ts`（动态列表与重复元素处理）。
* **核心技能** ：学习如何在浏览器中提取精准的 HTML 片段喂给 AI。

### 第三阶段：自然语言生成测试用例

* **任务** ：编写 `e2e.spec.ts`。
* **核心技能** ：将 POM 类定义“告诉” AI，用自然语言描述测试步骤，让 AI 组装代码。
#### 测试用例1：简单登录验证（基础级）

**业务场景**: 验证用户能够成功登录系统  
**测试步骤**:
1. 打开SwagLabs网站
2. 输入有效用户名和密码（standard_user / secret_sauce）
3. 点击登录按钮
4. 验证成功进入商品页面（页面标题显示"Products"）

**验证点**:
- ✅ 登录成功后页面跳转正确
- ✅ 页面显示商品列表标题

#### 测试用例2：完整购物流程验证（进阶级）

**业务场景**: 模拟用户完整的购物体验流程  
**测试步骤**:
1. 用户登录系统
2. 浏览商品页面，添加两件不同商品到购物车
   - 添加 "Sauce Labs Backpack"
   - 添加 "Sauce Labs Bike Light"
3. 验证购物车图标显示正确的商品数量（2）
4. 进入购物车页面，验证商品信息正确
5. 点击结账，填写用户信息
   - First Name: "John"
   - Last Name: "Doe"  
   - Postal Code: "12345"
6. 进入订单确认页面，验证商品和价格信息
7. 完成订单，验证成功页面显示

**验证点**:
- ✅ 商品成功添加到购物车
- ✅ 购物车数量徽章显示正确
- ✅ 购物车页面商品信息准确
- ✅ 结账表单正常提交
- ✅ 订单总价计算正确
- ✅ 订单完成确认信息显示
### 第四阶段：AI 调试与修复 

* **任务** ：人为制造一个 Bug（如修改定位器），利用报错日志让 AI 分析并修复。
* **挑战作业** ：实现“商品排序”功能的测试与验证。

## ⚠️ 关键注意事项 (避坑指南)

1. **“喂食”技巧** ：不要复制整个网页的 HTML（几千行）。只复制包含你目标元素的 **最小父级容器** （例如 `<form>` 或 `<div class="inventory_list">`），这样 AI 生成更准，Token 消耗更少。
2. **人工审查 (Code Review)** ：AI 生成的代码必须看一遍。

* 检查 Locator 是否合理（有没有用了看起来很奇怪的随机 ID？）。
* 检查 import 路径是否正确。

1. **数据隔离** ：不要把公司的真实敏感数据（密码、Key）发给公有大模型。在练习中使用 Swag Labs 的公开测试账号是安全的。
