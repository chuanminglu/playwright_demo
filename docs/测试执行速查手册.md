# Playwright 测试执行速查手册

## 📋 测试文件清单

```
tests/
├── e2e.spec.ts           # 完整购物流程测试（登录→添加购物车→结账）
├── e2e.login.spec.ts     # 用户登录功能测试
└── test-1.spec.ts        # 其他测试用例
```

---

## 🚀 常用执行命令

### 1. 运行所有测试

```bash
# 无头模式（默认，不显示浏览器）
npm run test

# 有头模式（显示浏览器窗口）
npm run test:headed

# UI 模式（交互式界面）
npm run test:ui
```

---

### 2. 运行指定测试文件

#### 运行完整购物流程测试
```bash
# 无头模式
npm run test -- tests/e2e.spec.ts

# 有头模式
npm run test -- tests/e2e.spec.ts --headed

# 调试模式
npm run test -- tests/e2e.spec.ts --debug
```

#### 运行登录测试
```bash
# 无头模式
npm run test -- tests/e2e.login.spec.ts

# 有头模式
npm run test -- tests/e2e.login.spec.ts --headed

# 调试模式
npm run test -- tests/e2e.login.spec.ts --debug
```

#### 运行 test-1 测试
```bash
# 无头模式
npm run test -- tests/test-1.spec.ts

# 有头模式
npm run test -- tests/test-1.spec.ts --headed
```

---

### 3. 运行指定测试用例（test case）

```bash
# 运行名称包含特定关键字的测试
npm run test -- tests/e2e.spec.ts --grep "purchase an item"

# 运行完整购物流程中的特定步骤
npm run test -- tests/e2e.spec.ts --grep "should login successfully" --headed
```

## 🎯 高级选项

### 指定浏览器

```bash
# 仅在 Chromium 运行
npm run test -- --project=chromium

# 仅在 Firefox 运行
npm run test -- --project=firefox

# 仅在 WebKit (Safari) 运行
npm run test -- --project=webkit
```

### 并发控制

```bash
# 串行运行（一个接一个）
npm run test -- --workers=1

# 并行运行（指定 worker 数量）
npm run test -- --workers=4
```

### 重试机制

```bash
# 失败时重试 2 次
npm run test -- --retries=2

# 仅重试失败的测试
npm run test -- --last-failed
```

### 调试相关

```bash
# 调试模式（逐步执行）
npm run test -- tests/e2e.spec.ts --debug

# 生成追踪文件
npm run test -- tests/e2e.spec.ts --trace on

# 查看测试报告
npx playwright show-report
```

## 📊 报告查看

### 查看 HTML 报告

```bash
# 生成并打开 HTML 报告
npx playwright show-report

# 如果报告已生成，直接打开
npx playwright show-report playwright-report
```

### 查看追踪文件

```bash
# 查看最新的追踪文件
npx playwright show-trace test-results/[test-name]/trace.zip
```

## 🔧 常见场景示例

### 场景 1：快速验证登录功能
```bash
npm run test -- tests/e2e.login.spec.ts --headed
```

### 场景 2：调试购物流程失败问题
```bash
npm run test -- tests/e2e.spec.ts --debug --project=chromium
```

### 场景 3：生成完整测试报告
```bash
npm run test -- --reporter=html
npx playwright show-report
```

### 场景 4：在多个浏览器运行所有测试
```bash
npm run test -- --project=chromium --project=firefox --project=webkit
```

### 场景 5：持续集成环境（CI）
```bash
npm run test -- --reporter=junit --reporter=html
```

## 📝 命令语法说明

### 基本格式
```
npm run test -- [测试文件路径] [选项]
```

### 参数说明

| 参数 | 说明 | 示例 |
|------|------|------|
| `--headed` | 显示浏览器窗口 | `npm run test -- --headed` |
| `--debug` | 调试模式（暂停执行） | `npm run test -- --debug` |
| `--ui` | UI 交互模式 | `npm run test:ui` |
| `--grep "关键字"` | 运行匹配的测试 | `npm run test -- --grep "login"` |
| `--project=浏览器` | 指定浏览器 | `npm run test -- --project=chromium` |
| `--workers=数量` | 并发数 | `npm run test -- --workers=1` |
| `--retries=次数` | 重试次数 | `npm run test -- --retries=2` |
| `--trace on` | 开启追踪 | `npm run test -- --trace on` |
| `--reporter=类型` | 报告类型 | `npm run test -- --reporter=html` |

## 🆘 故障排查

### 问题 1：测试超时
```bash
# 增加超时时间（在 playwright.config.ts 中配置）
# 或在命令中指定
npm run test -- --timeout=60000
```

### 问题 2：页面元素找不到
```bash
# 使用调试模式逐步检查
npm run test -- tests/e2e.spec.ts --debug

# 使用 UI 模式查看元素
npm run test:ui
```

### 问题 3：浏览器安装问题
```bash
# 重新安装 Playwright 浏览器
npx playwright install

# 仅安装 Chromium
npx playwright install chromium
```
