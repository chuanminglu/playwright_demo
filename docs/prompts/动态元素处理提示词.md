## 🎯 G - 目标与意图

### 核心目标

生成稳定可靠的动态元素定位方法，支持根据业务数据（文本、属性值等）精确操作目标元素，确保在数据变化时测试依然稳定。

### 具体目标

1. **精确定位**: 基于Playwright的文本过滤、属性匹配等功能实现精确元素定位
2. **参数化设计**: 支持动态传参，一个方法适配多种业务数据场景
3. **容错处理**: 考虑元素不存在、加载延迟等异常情况的处理
4. **性能优化**: 使用高效的选择器策略，避免过度复杂的DOM遍历

### 业务价值

- **为测试脚本**: 提供灵活的动态元素操作能力，支持数据驱动测试
- **为维护效率**: 一个方法处理多种相似操作，减少代码重复
- **为测试稳定性**: 基于业务语义而非DOM结构定位，提高测试抗变化能力
- **为扩展性**: 支持新增类似功能而无需大幅修改现有代码

### 成功标准

- ✅ 方法能够根据传入参数准确定位目标元素
- ✅ 支持常见的动态定位场景（文本匹配、属性过滤、索引选择）
- ✅ 包含合适的错误处理和等待策略
- ✅ 代码清晰易懂，便于后续维护和扩展

### 2. 质量要求

#### 定位策略优先级
- 第一优先级：`page.getByText()` 或 `page.getByRole()` 语义化定位
- 第二优先级：`filter()` 方法结合属性或文本过滤
- 第三优先级：动态CSS选择器或XPath（谨慎使用）

#### 参数化设计
- 支持字符串参数传入动态内容
- 考虑可选参数处理不同业务场景
- 参数验证确保输入有效性

#### 稳定性保障
- 使用适当的等待策略（waitFor、expect等）
- 避免硬编码的延时等待
- 考虑元素状态变化的影响

### 3. 格式规范

- **代码风格**: 与现有POM类保持一致
- **注释规范**: 添加方法功能和参数说明
- **错误处理**: 使用try-catch或Playwright内置错误处理

### 4. 特别说明

#### 常见动态定位模式

**模式1：文本内容匹配**
```typescript
// 根据文本内容找到对应的操作按钮
async clickButtonByText(targetText: string) {
  await this.page.getByText(targetText).locator('xpath=../button').click();
}
```

**模式2：属性值过滤**
```typescript
// 根据data-product-name属性值定位
async addItemByName(productName: string) {
  await this.page.locator(`[data-product-name="${productName}"] .add-btn`).click();
}
```

**模式3：组合定位**
```typescript
// 结合多个条件进行精确定位
async selectRowByValue(columnName: string, value: string) {
  const row = this.page.locator('tr').filter({ has: this.page.getByText(value) });
  await row.locator('.select-checkbox').click();
}
```

#### 性能考虑
- 避免使用过于复杂的选择器
- 优先使用Playwright内置的定位方法
- 合理使用缓存避免重复查找
