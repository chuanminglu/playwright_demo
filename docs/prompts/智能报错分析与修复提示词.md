## 📋 T - 任务描述

基于Playwright测试失败的错误日志和相关代码，分析故障根本原因，并提供具体的修复方案和优化建议。

### 输入材料

#### 材料1：错误日志信息

{粘贴完整的终端错误输出，包括：

{提供导致错误的具体代码段，包括：

#### 材料3：运行环境信息（可选）

{包括浏览器版本、操作系统、Playwright版本等环境信息}

### 任务上下文

本任务专门处理Playwright自动化测试过程中遇到的各类技术故障，需要快速定位问题根源并提供可执行的解决方案。

## 🎯 G - 目标与意图

### 核心目标

快速准确地识别Playwright测试失败的根本原因，提供可操作的修复方案，并给出预防类似问题的最佳实践建议。

### 具体目标

1. **问题诊断**: 基于错误信息准确判断故障类型和影响范围
2. **根因分析**: 深入分析导致问题的技术原因和业务背景
3. **解决方案**: 提供具体的代码修复建议和替代实现方案
4. **预防措施**: 给出避免类似问题重复出现的最佳实践建议

### 业务价值

- **为测试工程师**: 快速解决测试阻塞问题，提高问题解决效率
- **为项目进度**: 减少因测试失败导致的延期风险
- **为团队知识**: 积累故障处理经验，提升团队整体技术水平
- **为系统稳定**: 通过问题修复提升测试套件的可靠性

### 成功标准

- ✅ 准确识别错误的根本原因类型
- ✅ 提供可执行的具体修复代码
- ✅ 解释问题产生的技术背景和解决思路
- ✅ 给出预防类似问题的实践建议

## 📤 O - 输出要求

### 1. 输出结构

#### 第1部分：问题诊断
- 错误类型分类（元素定位失败、断言失败、超时等）
- 问题严重程度评估（P0-紧急、P1-高优先级、P2-中等、P3-低优先级）
- 影响范围分析（单个测试、测试套件、特定环境等）

#### 第2部分：根因分析  
- 技术层面的问题原因解释
- 可能的触发条件分析
- 与环境、数据、时序相关的因素

#### 第3部分：修复方案
- 具体的代码修改建议
- 替代实现方案（如果适用）
- 配置调整建议

#### 第4部分：预防措施
- 最佳实践建议
- 代码改进方向
- 监控和预警机制

### 2. 质量要求

#### 准确性要求
- 基于错误信息进行精确的问题定位
- 避免泛泛而谈，提供针对性解决方案
- 确保修复建议的可执行性

#### 完整性要求  
- 覆盖问题的技术和业务两个层面
- 提供短期修复和长期优化两种思路
- 考虑修复方案的副作用和影响

#### 实用性要求
- 修复代码可以直接复制粘贴使用
- 解释清晰易懂，便于学习和传播
- 提供验证修复效果的方法

### 3. 格式规范

- **结构化输出**: 使用标题和列表组织内容
- **代码高亮**: 使用代码块展示修复方案
- **优先级标记**: 使用emoji和标记突出重要信息
- **链接引用**: 提供相关文档和最佳实践的链接

### 4. 特别说明

#### 常见问题类型处理

**元素定位问题**
- 分析定位器失效原因（DOM变化、动态内容等）
- 提供更稳定的定位策略
- 建议使用多层定位保障

**时序和异步问题**
- 识别竞态条件和时序依赖
- 优化等待策略和超时设置
- 建议合适的同步机制

**环境和数据问题**
- 分析环境差异导致的失败
- 处理测试数据依赖和冲突
- 提供环境隔离和数据清理方案

#### 修复验证建议
每个修复方案应该包含：
1. 修复后的验证步骤
2. 可能的测试用例补充
3. 持续监控的建议

### 5. 输出格式示例

```markdown
## 🔍 问题诊断
**错误类型**: 元素定位失败
**严重程度**: P1-高优先级  
**影响范围**: LoginPage相关的所有测试用例

## 🎯 根因分析
页面DOM结构发生变化，原定位器`#login-button`不再有效...

## 🔧 修复方案
### 方案1：更新定位器（推荐）
```typescript
// 修改前
this.loginButton = page.locator('#login-button');

// 修改后  
this.loginButton = page.locator('[data-test="login-button"]');
```

## 🛡️ 预防措施
1. 优先使用data-test属性进行元素定位
2. 建立定位器健康度监控机制
3. 与开发团队协商DOM变更通知机制
```
