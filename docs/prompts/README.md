## 🔄 使用流程建议

### 阶段1：页面对象建模
1. 在浏览器开发者工具中复制目标页面的HTML片段
2. 使用**生成页面对象模型提示词**创建基础POM类
3. 如有动态元素需求，使用**动态元素处理提示词**补充方法

### 阶段2：测试用例开发  
1. 整理POM类的可用方法列表
2. 用自然语言描述测试业务流程
3. 使用**生成测试用例脚本提示词**组装完整测试

### 阶段3：问题诊断修复
1. 收集完整的错误日志和相关代码
2. 使用**智能报错分析与修复提示词**进行问题诊断
3. 按照修复建议优化测试代码

## 🎯 RTGO框架应用说明

所有提示词严格遵循RTGO框架：

- **R (Role)**: 定义专业的测试工程师角色，明确技术背景和擅长领域
- **T (Task)**: 清晰描述任务输入、输出和执行上下文
- **G (Goal)**: 阐明核心目标、具体目标、业务价值和成功标准  
- **O (Objective)**: 详细规定输出结构、质量要求、格式规范和特殊处理

## 💡 最佳实践提醒

### 输入准备
- **HTML片段**: 只复制包含目标元素的最小容器，避免整页HTML
- **错误日志**: 提供完整的错误堆栈，包含行号和具体报错信息
- **代码上下文**: 提供出错代码的前后文，便于准确定位问题

### 代码审查
- **定位策略**: 优先检查是否使用了稳定的data-test属性
- **异步处理**: 确保使用了合适的等待机制，避免硬编码延时
- **类型安全**: 验证TypeScript类型声明的正确性

### 维护优化
- **代码复用**: 及时提取公共方法，避免重复代码
- **错误处理**: 添加适当的异常处理和重试机制
- **文档更新**: 保持POM类方法的注释和文档同步
