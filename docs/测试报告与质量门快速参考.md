# Playwright测试报告与质量门快速参考

## 📊 支持的测试报告格式

### 1. HTML报告（默认）- 适合人工查看
```bash
npx playwright test
npx playwright show-report
```
✅ 可视化展示  
✅ 包含截图和视频  
✅ 支持交互式查看

---

### 2. JSON报告 - 适合CI/CD
```typescript
reporter: [['json', { outputFile: 'test-results/results.json' }]]
```
✅ 结构化数据  
✅ 程序化分析  
✅ 趋势分析

---

### 3. JUnit XML - 适合Jenkins
```typescript
reporter: [['junit', { outputFile: 'test-results/junit.xml' }]]
```
✅ 标准格式  
✅ Jenkins原生支持  
✅ 生成趋势图

---

### 4. GitHub Actions原生报告
```typescript
reporter: [['github']]
```
✅ GitHub集成  
✅ PR中显示错误  
✅ 自动Annotations

---

### 5. Allure报告（企业级）
```bash
npm install -D allure-playwright
```
✅ 企业级可视化  
✅ 历史趋势  
✅ 用例分类

## 🎯 质量门集成方案

### 本地测试 + 质量门
```bash
npm run test:ci
```
执行流程：
1. 运行所有Playwright测试
2. 生成JSON报告
3. 执行质量门检查脚本
4. 输出通过/失败结果

---

### CI/CD集成（GitHub Actions）

**文件位置**: `.github/workflows/playwright-ci.yml`

**触发条件**:
- Push到main/develop分支
- Pull Request到main分支

**执行步骤**:
1. 安装依赖和浏览器
2. 运行Playwright测试
3. 上传测试报告（Artifact）
4. 质量门评估
5. PR评论测试结果（可选）

---

## 📋 质量门标准

| 指标 | 标准 | 说明 |
|------|------|------|
| 测试通过率 | 100% | 所有测试必须通过 |
| Flaky测试 | 0 | 不允许不稳定的测试 |
| 执行时间 | < 5分钟 | 整体测试执行时间 |

---

## 🔧 配置分支保护（必须）

### 步骤1：进入仓库设置
```
Settings → Branches → Add rule
```

### 步骤2：配置保护规则
```
Branch name pattern: main

☑️ Require status checks to pass before merging
  ☑️ E2E测试质量门 (job名称)
  
☑️ Do not allow bypassing the above settings
```

### 效果
```
PR合并时必须等待质量门检查通过
测试失败 → 🚫 阻止合并
测试通过 → ✅ 允许合并
```

## 📈 查看测试报告的方式

### 1. 本地查看HTML报告
```bash

```

- 选择workflow运行
- 展开测试步骤查看日志

```

在workflow中配置自动发布，访问：
```
https://<username>.github.io/<repo-name>/

## ✅ 完整工作流

### 开发阶段
```bash
# 1. 创建功能分支
git checkout -b feature/new-test

# 2. 编写测试
# pages/NewPage.ts
# tests/new.spec.ts

# 3. 本地运行测试
npm run test:ci

# 4. 提交代码
git add .
git commit -m "feat: add new E2E test"
git push origin feature/new-test
```

### PR阶段
```
1. 在GitHub创建PR: feature/new-test → main
2. GitHub自动触发CI
3. 等待质量门检查通过
4. 通过后合并PR
```

### 质量门结果
```
✅ 通过 → 允许合并
❌ 失败 → 修复后重新推送
```

## 🔍 常见问题

### Q1: 如何查看失败测试的详细信息？
```bash
# 本地查看HTML报告
npx playwright show-report

# CI中下载Artifact
Actions → workflow详情 → Artifacts → playwright-report
```

### Q2: 测试在CI失败但本地通过？
```typescript
// 增加超时时间
timeout: 30000,

// 添加重试
retries: process.env.CI ? 2 : 0,

// 等待网络空闲
await page.waitForLoadState('networkidle');
```

### Q3: 如何临时禁用质量门？
```typescript
// 测试文件中跳过测试
test.skip('临时跳过的测试', async ({ page }) => {
  // ...
});

// 或者在CI中
continue-on-error: true  # 不推荐！
```
